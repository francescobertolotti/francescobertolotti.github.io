Implementation Rules

  • Usare PptxGenJS in modo obbligatorio per implementare il deck.
  • Rispettare stile del template: tipografia, spacing, palette, layout.
  • Eccezione rapida: python-pptx o artifact_tool solo per quick edits triviali.
  • Importare helper ufficiali, senza copiare/incollare il loro codice nei deliverable.
  • Lavorare in working dir locale e copiare artefatti solo dopo validazione finale.

Container Tools

  • ensure_raster_image.py: rasterizza/conv. in PNG.
  • render_slides.py: render PPTX in PNG (slide-1.png, slide-2.png...).
  • create_montage.py: montage tiled da immagini di una cartella.
  • slides_test.py: rileva overflow fuori canvas.
  • Eseguire sempre con -h per opzioni complete.

Helpers API Highlights

  • autoFontSize, calcTextBox, calcTextBoxHeightSimple per gestione testo.
  • imageSizingCrop e imageSizingContain per immagini sicure.
  • svgToDataUri e latexToSvgDataUri per contenuti vettoriali.
  • getImageDimensions, safeOuterShadow, codeToRuns.
  • alignSlideElements e distributeSlideElements per allineamento rigoroso.

Page Size & Layout

  • Default: 16:9 (13 1/3 x 7.5 in, LAYOUT_WIDE).
  • Se richiesto da input/template, usare aspect ratio differente (es. 4:3).
  • Se non definito esplicitamente, derivare dimensioni con getSlideDimensions.
  • Per 16:9, preferire calcolo esatto: (7.5 / 9) * 16.

Visual Rules (Images)

  • Evitare sizing built-in in addImage (bug noti).
  • Preferire helper: crop default, contain per grafici/testi, stretch per texture.
  • Non combinare helper e campo sizing insieme.
  • Input supportati: path locale, data URI, SVG string, Buffer.
  • Mirroring orizzontale: flipH: true.

Quality Gates

  • Usare warnIfSlideHasOverlaps per overlap tra elementi.
  • Usare warnIfSlideElementsOutOfBounds per elementi fuori canvas.
  • Correggere tutti gli overlap severi e warning non intenzionali prima del delivery.
  • Impostare margini/griglia coerenti e spaziatura uniforme tra peer elements.
  • Rilanciare diagnostica dopo ogni passaggio di layout importante.

Workflow Rapido Consigliato

1) Definisci aspect ratio e griglia coerente.

2) Inserisci testo con helper di fit e gerarchia tipografica.

3) Posiziona media con crop/contain appropriati.

4) Esegui warning checks e correggi overlap/out-of-bounds.

5) Aggiungi note con blocchi [Sources] per fonti esterne.

Web Search & Sources

  • Dopo download immagine, validare con comando file.
  • Le thumbnail possono essere center-cropped: verificare aspect ratio reale.
  • Per claim non banali e asset esterni, includere sempre [Sources] nelle speaker notes.
